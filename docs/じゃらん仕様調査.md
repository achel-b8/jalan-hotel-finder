# じゃらんnet 仕様調査（現行仕様 v1）

更新日: 2026-03-05
対象: `jalan-hotel-finder` v1

## 1. 文書目的
- 本書は、じゃらん調査結果を「時系列メモ」ではなく「現行実装仕様」として固定するための仕様書である。
- `area` / `list` / `coupon` の3ユースケースで、v1で採用するURL構造・パラメータ・抽出方式・例外判定を定義する。
- 詳細な受け入れ要件とCLI I/Fは `docs/CLI仕様書.md` を正本とし、本書はその技術根拠を扱う。

## 2. 対象ユースケース
- `area`: 都道府県配下のエリアを横断して宿一覧を取得する。
- `list`: 候補宿名CSVを起点に keyword one-shot 検索して宿を抽出する。
- `coupon`: クーポン名を `couponId` に解決し、対象宿を取得する。

## 3. URL・ルーティング仕様

### 3.1 エリア検索 (`area`)
- URL形式: `https://www.jalan.net/{pref_code}/LRG_{lrg_digits}/SML_{sml_digits}/`
- 実行単位は `SML` 単体ではなく `pref/lrg/sml` のルートタプル。
- ルートは同梱 `area.xml` から展開し、後述の「再配置補正」と「固定除外」を適用して確定する。
- 主要パラメータ:
  - `stayYear`, `stayMonth`, `stayDay`
  - `adultNum`, `stayCount`, `roomCount=1`
  - `dateUndecided=0`, `careBath=0`
  - 任意: `mealType`, `maxPrice`, `careKake`, `careBathRent`, `carePribateBath`

### 3.2 候補宿検索 (`list`)
- URL形式: `https://www.jalan.net/uw/uwp2011/uww2011init.do?...`
- `list` は keyword one-shot 固定（1候補名につき1リクエスト、ページ送りなし）。
- `keyword` は CP932 `%xx` エンコードを採用する。
- 固定パラメータ:
  - `distCd=06`, `rootCd=7701`, `screenId=FWPCTOP`, `ccnt=button-fw`, `image1=`
  - `roomCount=1`, `dateUndecided=0`, `careBath=0`
- 候補CSV `優先オプション` は次へ写像:
  - `care-kakenagashi` -> `careKake=1`
  - `care-bath-rent` -> `careBathRent=1`
  - `care-private-openair` -> `carePribateBath=1`

### 3.3 クーポン検索 (`coupon`)
- URL形式: `https://www.jalan.net/uw/uwp1400/uww1405.do?...`
- `coupon_source_url` から `couponName -> couponId` を解決し、都道府県配下 `LRG` 単位で検索する。
- 必須クエリ:
  - `screenId=UWW7801`, `couponId`, `kenCd`, `lrgCd`, `idx`
  - `stayYear`, `stayMonth`, `stayDay`, `stayCount`
  - `adultNum`, `roomCount=1`, `roomCrack=<adults>00000`
  - `rootCd=`, `afCd=`

### 3.4 主要マッピングの理由と目的
- v1では「ユーザー入力の意味」を、じゃらん側で実際に解釈されるクエリへ明示的に写像する。
- 暗黙推定を避け、同一入力で同一URLを再現できることを優先する。

| マッピング対象 | マッピング内容 | なぜこの値にするか | 目的 |
|---|---|---|---|
| エリアルート | `pref/lrg/sml` タプルをURLへ反映 | `SML` だけでは誤った `LRG` を組み立てるケースがあるため | 404/エラーテンプレートの回避、検索漏れ防止 |
| 宿名keyword | 候補宿名 -> CP932 `%xx` エンコード | 日本語keywordでCP932前提のページが存在するため | 0件化リスクの低減、`list` 安定化 |
| `meal_type` | `none/breakfast/dinner/two_meals` -> `mealType=0/1/2/3` | じゃらん側の列挙値に一致させる必要があるため | CLI入力と検索条件の一貫性確保 |
| `優先オプション` | `care-*` トークン -> `careKake/careBathRent/carePribateBath=1` | 候補CSVの人間可読トークンをURL条件へ落とし込むため | 候補宿ごとの条件差を機械的に再現 |
| `coupon` 人数 | `adults` -> `adultNum` と `roomCrack=<adults>00000` | `uww1405` で必要な人数関連クエリのため | クーポン検索結果の整合性維持 |

## 4. area.xml 運用仕様（現行）
- 参照元: 同梱スナップショット `src/jalan_hotel_finder/infrastructure/data/area.xml`
- 実行時に外部 `area.xml` を毎回取得しない。
- `area` 検索のルートは `Prefecture -> LargeArea -> SmallArea` から `AreaRoute(pref_code, lrg_code, sml_code, ...)` を生成して確定する。

### 4.1 ルート確定ルール（実装）
`resolve_area_routes_for_prefecture` は次の順序で検索対象ルートを確定する。
1. 都道府県配下の `SmallArea` から `SML` 値を列挙する。
2. 固定除外SMLに含まれる場合は候補から除外する。
3. 固定再配置マップに `SML` が存在する場合は `LRG` を補正する。
4. `SML` 重複を除去し、`AreaRoute` 一覧として返す。

### 4.3 固定除外SML（現行）
現行の固定除外は `SML_251105`（滋賀県 / 伊香）のみ。

| SMLコード | 都道府県 | `area.xml` 上のLRG | 判定 |
|---|---|---|---|
| `SML_251105` | 滋賀県 | `LRG_251100`（奥琵琶湖・湖西） | `.../SML_251105/` が `エラー画面` かつ宿リンク0件 |

補足:
- `LRG_251100` 自体は正常応答だが、`SML_251105` 個別ルートは live上で非公開/無効の状態。
- 現行実装では `SML` 単位検索を前提にするため、`SML_251105` は除外維持とする。

## 5. ページネーション仕様
- `area`: `idx=0,30,60...` を追跡し、次ページがなくなるまで巡回。
- `coupon`: `idx=0,30,60...` を同様に追跡。
- `list`: one-shot 方針のため `dispStartIndex` は追跡しない。

## 6. 抽出仕様（出力4項目）
- 必須抽出項目:
  - `hotel_name`
  - `hotel_url`
  - `plan_name`
  - `price`
- 抽出器は現行DOMを主対象とし、`openYadoSyosai('NNNNNN', ...)` 形式のリンクも解釈する。
- `coupon` (`uww1405`) では `yadNo` を `/yadxxxxxx/` へ正規化して同一宿判定に利用する。

## 7. 正規化・重複排除仕様
- 同一宿判定は `hotel_url` のクエリを除いたURLパス単位で実施する。
- 同一宿の保持上限は最大3プラン（取得順）とする。

## 8. 異常系判定仕様

### 8.1 アクセス制限判定 (`AccessRestrictedError`)
- 次のいずれかでアクセス制限とみなす:
  - HTTP `403` / `429`
  - 制限キーワードを含み、かつ宿リンク (`/yad`) が0件
- 制限キーワード（v1）:
  - `アクセスが集中`
  - `ただいまアクセスが集中`
  - `ご利用を制限`
  - `しばらく時間をおいて`
  - `短時間に多くのアクセス`
  - `不正なアクセス`
  - `Too Many Requests`
  - `Forbidden`

### 8.2 無効エリアルート判定 (`InvalidAreaRouteError`)
- `/{pref}/LRG_xxxxxx/SML_xxxxxx/` 形式URLで、次の全条件を満たす場合は「無効ルート疑い」とする:
  - HTTP `200`
  - 宿リンク (`/yad`) が0件
  - 本文に `エラー画面` を含む
  - 本文に `ただ今の時間帯アクセスが集中しているため` を含む

### 8.3 失敗ポリシー
- v1は `stop` 固定。
- 1エリア（または1検索単位）失敗時点で処理を中断し、取得失敗として扱う。

## 9. ユースケース別の確定仕様

### 9.1 `area`
- `--pref` 指定時は都道府県ごとに `AreaRoute` を展開する。
- 展開時に再配置補正・固定除外を適用する。
- 検索結果はページ巡回後に重複排除し、最大3プラン/宿で出力する。

### 9.2 `list`
- 候補CSVは `data/candidate_hotels.csv` 固定。
- 候補宿名ごとに keyword URL を1回だけ実行する。
- 宿名部分一致または宿URL一致で照合し、重複排除して出力する。

### 9.3 `coupon`
- `coupon_name` を完全一致で `couponId` に解決する。
- 都道府県配下 `LRG` を展開して `uww1405` を巡回する。
- 出力は `area/list` と同一フォーマットを維持する。

## 10. 既知制約
- `careBath=1` は「内湯・大浴場」であり、客室内風呂の保証条件ではない。
- 「客室に内風呂あり」を直接保証する公開パラメータは確認できていない。
- `list` は網羅性より速度と安定性を優先し、one-shot 固定とする。
